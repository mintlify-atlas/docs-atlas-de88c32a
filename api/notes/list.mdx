## Endpoint

```
GET /api/notes
```

Retrieves all notes belonging to the authenticated user, ordered by most recently updated first.

## Authentication

This endpoint requires authentication via Clerk. The user must have a valid session.

## Rate Limiting

This is a **read operation** with a rate limit of **30 requests per 10 seconds** per user.

## Request

No parameters required. The endpoint automatically filters notes by the authenticated user's ID.

### Example Request

```bash
curl -X GET https://your-domain.com/api/notes \
  -H "Cookie: __session=your-session-token"
```

## Implementation Details

### Database Query

The endpoint executes the following query:

```typescript
db.select()
  .from(notes)
  .where(eq(notes.userId, userId))
  .orderBy(desc(notes.updatedAt))
```

### Sorting

Notes are automatically sorted by `updatedAt` in descending order (most recently updated first). This ensures that the user's most active notes appear at the top of the list.

### Performance

The database includes an index on `userId` (`user_id_idx`) to optimize query performance for retrieving user-specific notes.
