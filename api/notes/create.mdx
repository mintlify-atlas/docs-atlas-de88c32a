## Endpoint

```
POST /api/notes
```

Creates a new note for the authenticated user with validation.

## Authentication

This endpoint requires authentication via Clerk. The user must have a valid session.

## Rate Limiting

This is a **write operation** with a rate limit of **10 requests per 10 seconds** per user.

## Request

### Body Parameters

<ParamField path="title" type="string" required>
  Title of the note
  
  **Validation:**
  - Minimum: 3 characters
  - Maximum: 100 characters
  - Automatically trimmed
</ParamField>

<ParamField path="textNote" type="string" required>
  Content of the note
  
  **Validation:**
  - Minimum: 3 characters
  - Maximum: 400 characters
  - Automatically trimmed
</ParamField>

### Example Request

```bash
curl -X POST https://your-domain.com/api/notes \
  -H "Content-Type: application/json" \
  -H "Cookie: __session=your-session-token" \
  -d '{
    "title": "Project Ideas",
    "textNote": "Brainstorm new features for the Q2 roadmap including mobile app support and API improvements."
  }'
```

## Error Responses

### Unauthorized (401)

Returned when the request is not authenticated:

```json
{
  "error": "No autenticado"
}
```

### Validation Error (400)

Returned when the request body fails validation. The `details` field contains field-specific error messages:

```json
{
  "error": "Datos inválidos",
  "details": {
    "title": ["String must contain at least 3 character(s)"],
    "textNote": ["String must contain at least 3 character(s)"]
  }
}
```

### Rate Limit Exceeded (429)

Returned when the user exceeds 10 write requests per 10 seconds:

```json
{
  "error": "Demasiadas solicitudes, intenta en unos segundos",
  "retryAfter": 6
}
```

Headers include:
- `Retry-After`: Seconds until the rate limit resets

### Server Error (500)

Returned when an unexpected server error occurs:

```json
{
  "error": "Error del servidor"
}
```

## Implementation Details

### Validation

The endpoint uses Zod for schema validation with the `createNoteSchema`:

```typescript
const createNoteSchema = z.object({
  title: z.string().min(3).max(100).trim(),
  textNote: z.string().min(3).max(400).trim(),
})
```

### Database Insert

After validation, the note is inserted into the database:

```typescript
db.insert(notes)
  .values({
    title: validation.data.title,
    textNote: validation.data.textNote,
    userId: userId,
  })
  .returning()
```

The `id`, `createdAt`, and `updatedAt` fields are automatically generated by the database.

### Automatic Fields

- `id`: Auto-generated UUID v4
- `userId`: Set from authenticated user's Clerk ID
- `createdAt`: Set to current Unix timestamp
- `updatedAt`: Set to current Unix timestamp
