## Protected Routes

Pad Notes automatically protects your sensitive pages and API endpoints. The middleware ensures you're authenticated before accessing:

- `/dashboard` - Your personal notes workspace
- `/api/notes` - All note management endpoints

```typescript
// From: client/src/middleware.ts:3
const isProtectedRoute = createRouteMatcher(['/dashboard', '/api/notes(.*)'])

export const onRequest = clerkMiddleware((auth, context) => {
  const { userId, redirectToSignIn } = auth()
  const { pathname } = new URL(context.request.url)

  if (userId && pathname === '/') {
    return context.redirect('/dashboard')
  }

  if (isProtectedRoute(context.request) && !userId) {
    return redirectToSignIn()
  }
})
```

<Info>
  If you're already signed in and visit the homepage, you'll be automatically redirected to your dashboard.
</Info>

## API Authentication

Every API request is authenticated server-side. Here's how it works:

```typescript
// From: client/src/pages/api/notes/index.ts:9
export const GET: APIRoute = async ({ locals }) => {
  const { userId, isAuthenticated } = locals.auth()

  if (!isAuthenticated) {
    return new Response(JSON.stringify({ error: 'No autenticado' }), {
      status: 401,
      headers: { 'Content-Type': 'application/json' },
    })
  }
  // ... fetch user's notes
}
```

<Warning>
  All API endpoints return a `401 Unauthorized` response if you're not authenticated. The frontend will prompt you to sign in again.
</Warning>

## User Isolation

Your notes are completely private. The database schema ensures notes are tied to your unique user ID:

```typescript
// From: client/src/db/schema.ts:4
export const notes = sqliteTable(
  'notes',
  {
    id: text('id').primaryKey(),
    title: text('title').notNull(),
    textNote: text('text_note').notNull(),
    userId: text('user_id').notNull(),  // Your unique Clerk user ID
    // ...
  },
  (table) => ({
    userIdIdx: index('user_id_idx').on(table.userId),  // Indexed for fast queries
  })
)
```

Every database query filters by your `userId`, ensuring you only see your own notes.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't access my dashboard">
    If you're redirected to sign in when trying to access your dashboard:
    1. Your session may have expired
    2. Clear your browser cookies and sign in again
    3. Ensure cookies are enabled in your browser
  </Accordion>
  
  <Accordion title="Google OAuth not working">
    If Google sign-in fails:
    1. Ensure you're not blocking third-party cookies
    2. Try a different browser
    3. Check that you have a stable internet connection
  </Accordion>
  
  <Accordion title="API returns 401 errors">
    If you're seeing authentication errors:
    1. Sign out and sign in again to refresh your session
    2. Check if your session has expired
    3. Clear your browser cache and cookies
  </Accordion>
</AccordionGroup>

<Info>
  Pad Notes never stores your password directly. All authentication is handled securely by Clerk's infrastructure.
</Info>
