Pad Notes provides a clean, intuitive interface for managing your personal notes. Every note you create is automatically saved to the cloud and synced across all your devices.

## Creating Notes

### Validation Rules

Pad Notes enforces the following constraints to ensure data quality:

```typescript
// From: client/src/lib/validations.ts:4
export const createNoteSchema = z.object({

  textNote: z.string().min(3).max(400).trim(),
```

| Field | Minimum | Maximum | Required |
|-------|---------|---------|----------|
| **Title** | 3 characters | 100 characters | Yes |
| **Content** | 3 characters | 400 characters | Yes |

<Info>

</Info>

### Save Process

When you click **"Guardar"** (Save), here's what happens:

```jsx
// From: client/src/components/SendNotes.jsx:16
const handleSubmit = async (e) => {
  e.preventDefault()

  const response = await fetch('/api/notes', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      title: noteTitle,
      textNote: note,
    }),
  })

  if (response.status === 429) {
    toast.error({ text: 'Demasiadas solicitudes, intenta en unos segundos' })
    return
  }

  if (!response.ok) {
    toast.error({ text: data.error || 'Error al crear la nota' })
    return
  }

  toast.success({ text: 'Nota creada correctamente' })
}
```

<Warning>
  The "Save" button is disabled until both the title and content fields contain text.
</Warning>

## Database Storage

Notes are stored in a Turso (SQLite) database with the following schema:

```typescript
// From: client/src/db/schema.ts:4
export const notes = sqliteTable('notes', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  title: text('title').notNull(),
  textNote: text('text_note').notNull(),
  userId: text('user_id').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull().default(sql`(unixepoch())`),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull().default(sql`(unixepoch())`),
})
```

### Auto-Generated Fields

- **id**: Unique UUID generated automatically
- **userId**: Your Clerk user ID (ensures note privacy)
- **createdAt**: Unix timestamp when note was created
- **updatedAt**: Unix timestamp of last modification

### Note Ordering

Notes are sorted by most recently updated first:

```typescript
// From: client/src/pages/api/notes/index.ts:38
const userNotes = await db
  .select()
  .from(notes)
  .where(eq(notes.userId, userId))
  .orderBy(desc(notes.updatedAt))  // Most recent first
```

### Update Validation

Edits use a relaxed validation schema where both fields are optional:

```typescript
// From: client/src/lib/validations.ts:10
export const updateNoteSchema = z.object({
  title: z.string().min(3).max(100).trim().optional(),
  textNote: z.string().min(3).max(400).trim().optional(),
})
```

You can update just the title, just the content, or both.

### Server Update

```typescript
// From: client/src/pages/api/notes/[id].ts:68
const [updatedNote] = await db
  .update(notes)
  .set({
    ...validation.data,
    updatedAt: new Date(),  // Automatically updates timestamp
  })
  .where(and(eq(notes.id, params.id), eq(notes.userId, userId)))  // Ensures you own the note
  .returning()
```

<Warning>
  You can only edit notes that belong to you. The server verifies ownership before allowing updates.
</Warning>

<Warning>
  Deletions are permanent. There is no undo or trash/recycle bin feature.
</Warning>

### Hard Delete

Pad Notes performs a hard delete (no soft delete or archiving):

```typescript
// From: client/src/pages/api/notes/[id].ts:137
const [deletedNote] = await db
  .delete(notes)
  .where(and(eq(notes.id, params.id), eq(notes.userId, userId)))
  .returning()

if (!deletedNote) {
  return new Response(JSON.stringify({ error: 'Nota no encontrada' }), {
    status: 404,
  })
}
```

### Loading States

The UI shows different states based on your data:

```jsx
// From: client/src/components/ListNotes.jsx:27
const fetchNotes = async () => {
  try {
    setIsLoading(true)
    const response = await fetch('/api/notes')
    const data = await response.json()
    setNotes(data.data || [])
  } catch (error) {
    setError(error)
  } finally {
    setIsLoading(false)
  }
}
```

| State | Display |
|-------|--------|
| **Loading** | Spinner with "Cargando notas..." message |
| **Empty** | Sad face image with "No hay notas para mostrar" |
| **Error** | Error icon with error message |
| **Success** | Grid of note cards |

## Performance Optimization

### Indexed Queries

The database uses an index on `userId` for fast retrieval:

```typescript
// From: client/src/db/schema.ts:24
(table) => ({
  userIdIdx: index('user_id_idx').on(table.userId),
})
```

This ensures your notes load quickly, even if you have hundreds of them.

### Real-Time Updates

The note list automatically refreshes after any create, update, or delete operation:

```jsx
// From: client/src/hooks/useChangeNotes.jsx
const { refreshNotes, handleNoteAdded } = useChangeNotes()

useEffect(() => {
  fetchNotes()
}, [refreshNotes, fetchNotes])
```

<Info>
  You don't need to manually refresh the page. The UI updates automatically when you modify notes.
</Info>
