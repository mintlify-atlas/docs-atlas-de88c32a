Pad Notes includes a powerful, client-side search feature that helps you find notes instantly without waiting for server responses.

## Search Interface

### Opening Search

Click the **search icon** in your dashboard to reveal the search bar:

```jsx
// From: client/src/components/filterSearch.jsx:40
<button
  onClick={handleClick}
  className="rounded-md bg-cyan-600 px-1 py-0.5"
>
  <Search className='text-white'/>
</button>
```

The search bar animates smoothly into view with a slide-in effect:

```jsx
// From: client/src/components/filterSearch.jsx:28
<input
  type="text"
  className={`rounded-md bg-white py-1 pl-1 ${
    barIsVisible ? 'w-full opacity-100' : 'w-0 opacity-0'
  } transition-all duration-300 ease-in-out`}
  placeholder="Nota a buscar"
  onChange={(e) => setSearchText(e.target.value)}
  value={searchText}
/>
```

### Auto-Focus

When the search bar opens, it automatically receives focus so you can start typing immediately:

```jsx
// From: client/src/components/filterSearch.jsx:8
useEffect(() => {
  if (barIsVisible && inputRef.current) {
    inputRef.current.focus()
  }
}, [barIsVisible])
```

<Info>
  Press **Escape** to close the search bar and clear focus.
</Info>

## Real-Time Filtering

### Instant Results

Search happens on the client side as you type - no server requests needed:

```jsx
// From: client/src/components/NoteArticle.jsx:85
const filteredNotes = notes.filter((note) => {
  if (searchTerm === '') return true
  return (
    note.title.toLowerCase().includes(searchTerm.toLowerCase()) ||
    note.textNote.toLowerCase().includes(searchTerm.toLowerCase())
  )
})
```

## Match Highlighting

### Visual Feedback

Pad Notes highlights search matches in yellow, making them easy to spot:

```jsx
// From: client/src/components/NoteArticle.jsx:28
function highLightMatch(text, searchTerm) {
  if (!searchTerm || searchTerm === '') return text

  // Escape special regex characters
  const escapedSearchTerm = searchTerm.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')

  // Create case-insensitive global regex
  const regex = new RegExp(`(${escapedSearchTerm})`, 'gi')

  // Split and highlight matches
  return text.split(regex).map((part, i) => {
    if (part.toLowerCase() === searchTerm.toLowerCase()) {
      return (
        <mark key={i} className="rounded bg-yellow-200 px-0.5 dark:bg-yellow-700 dark:text-white">
          {part}
        </mark>
      )
    }
    return part
  })
}
```

### Highlighting Locations

Matches are highlighted in three places:

1. **Note Card Title** (grid view)
```jsx
// From: client/src/components/NoteArticle.jsx:133
<h2 className="self-start font-bold text-black uppercase md:text-xl dark:text-white">
  {highLightMatch(note.title, searchTerm)}
</h2>
```

2. **Note Card Preview** (grid view)
```jsx
// From: client/src/components/NoteArticle.jsx:136
<p className="mb-2.5 w-full self-start overflow-hidden text-sm text-ellipsis whitespace-nowrap">
  {highLightMatch(note.textNote.split('\n')[0], searchTerm)}
  {note.textNote.includes('\n') ? '...' : ''}
</p>
```

3. **Full Note Content** (modal view)
```jsx
// From: client/src/components/NoteArticle.jsx:209
<h2 className="self-start text-xl font-bold text-black uppercase dark:text-white">
  {highLightMatch(note.title, searchTerm)}
</h2>
<p className="my-4 self-start text-start break-words whitespace-pre-line text-black dark:text-white">
  {highLightMatch(note.textNote, searchTerm)}
</p>
```

The search function safely handles special characters that could break regex:


## Search Limitations

### Client-Side Only

Search only works on notes that are already loaded in your browser:

- All your notes are fetched on page load
- Search cannot find notes you haven't created yet
- Search doesn't work across multiple users (notes are private)

### Memory Considerations

Since search happens client-side, all notes must be in browser memory:

```jsx
// From: client/src/components/ListNotes.jsx:30
const response = await fetch('/api/notes')
const data = await response.json()
setNotes(data.data || [])  // All notes loaded at once
```

This works well for personal note-taking apps where users typically have dozens to hundreds of notes, not thousands.

### No Advanced Operators

The search currently doesn't support:
- Boolean operators (AND, OR, NOT)
- Exact phrase matching ("quotes")
- Wildcard characters (* or ?)
- Regular expression queries

<Warning>
  If you need to search very large note collections or use advanced search operators, consider requesting a server-side search feature.
</Warning>
