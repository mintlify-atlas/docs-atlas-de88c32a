Pad Notes is a modern web application designed to help you organize your ideas simply and quickly. Built with cutting-edge technologies, it provides a clean, minimalist experience for storing information, ideas, and everything that inspires your day-to-day life.

## What is Pad Notes?

Pad Notes is a full-stack SaaS application that combines the power of Astro's server-side rendering with React's interactivity, Clerk's robust authentication, and Turso's edge database to deliver a fast, secure, and scalable note-taking experience.

The application features:
- **Secure authentication** with Google OAuth and email/password support
- **Personal note management** with full CRUD operations
- **Rate limiting** to protect against abuse
- **Real-time search** with match highlighting
- **User isolation** ensuring each user only accesses their own notes
- **Responsive design** that works seamlessly across devices

## Architecture Overview

Pad Notes follows a modern full-stack architecture:

### Frontend Layer
- **Astro** - Server-side rendering and routing
- **React** - Interactive UI components
- **Tailwind CSS 4** - Utility-first styling with the latest features

### Backend Layer
- **Astro API Routes** - Serverless functions for CRUD operations
- **Drizzle ORM** - Type-safe database queries
- **Zod** - Runtime schema validation

### Data Layer
- **Turso (libSQL)** - Edge-first SQLite database
- **Upstash Redis** - Rate limiting and caching

### Authentication
- **Clerk** - Complete authentication solution with OAuth support

## Database Schema

The application uses a simple, efficient schema defined with Drizzle ORM:

```typescript src/db/schema.ts
import { sql } from 'drizzle-orm'
import { index, integer, sqliteTable, text } from 'drizzle-orm/sqlite-core'

export const notes = sqliteTable(
  'notes',
  {
    id: text('id')
      .primaryKey()
      .notNull()
      .$defaultFn(() => crypto.randomUUID()),
    title: text('title').notNull(),
    textNote: text('text_note').notNull(),
    userId: text('user_id').notNull(),
    createdAt: integer('created_at', { mode: 'timestamp' })
      .notNull()
      .default(sql`(unixepoch())`),
    updatedAt: integer('updated_at', { mode: 'timestamp' })
      .notNull()
      .default(sql`(unixepoch())`),
  },
  (table) => ({
    userIdIdx: index('user_id_idx').on(table.userId),
  }),
)
```

<Note>
  The `userId` field is indexed to optimize queries when filtering notes by user, ensuring fast performance even with many users.
</Note>

## Security Features

### User Isolation

All database queries include user ID verification to ensure users can only access their own notes:

```typescript src/pages/api/notes/index.ts
const userNotes = await db
  .select()
  .from(notes)
  .where(eq(notes.userId, userId))
  .orderBy(desc(notes.updatedAt))
```

### Rate Limiting

Protection against abuse is implemented using Upstash Redis with sliding window rate limits:

```typescript src/lib/ratelimit.ts
export const writeRatelimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(10, '10 s'),
  analytics: true,
  prefix: '@upstash/ratelimit/write',
})

export const readRatelimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(30, '10 s'),
  analytics: true,
  prefix: '@upstash/ratelimit/read',
})
```

### Middleware Protection

Routes are protected using Clerk's middleware:

```typescript src/middleware.ts
import { clerkMiddleware, createRouteMatcher } from '@clerk/astro/server'

const isProtectedRoute = createRouteMatcher(['/dashboard', '/api/notes(.*)'])

export const onRequest = clerkMiddleware((auth, context) => {
  const { userId, redirectToSignIn } = auth()
  
  if (isProtectedRoute(context.request) && !userId) {
    return redirectToSignIn()
  }
})
```
